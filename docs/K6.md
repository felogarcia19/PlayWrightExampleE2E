# K6 Performance & Load Testing Guide 📊

## Overview

K6 is an open-source load testing tool for performance and stress testing. It uses JavaScript/Go for writing tests.

**Location:** `/k6-performance/`

---

## 📋 Prerequisites

- K6 installed ([Installation Guide](https://k6.io/docs/getting-started/installation/))
- Node.js (optional, for scripting)
- Basic understanding of HTTP protocols

---

## 🚀 Installation

### macOS
```bash
brew install k6
```

### Linux
```bash
sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv-keys C5AD17C747E3415A3642D57D77C6C491D6AC1D69
echo "deb https://dl.k6.io/deb stable main" | sudo tee /etc/apt/sources.list.d/k6-stable.list
sudo apt-get update
sudo apt-get install k6
```

### Windows
```bash
choco install k6
```

## 🏃 Running Tests

### Run Basic Test
```bash
k6 run scripts/smoke-tests.js
```

### Run with Output Options
```bash
k6 run scripts/load-tests.js --out csv=results.csv
k6 run scripts/load-tests.js --out json=results.json
```

### Run with Custom Configuration
```bash
k6 run --vus 100 --duration 30s scripts/load-tests.js
```

### Run in Cloud
```bash
k6 run --cloud scripts/load-tests.js
```

---

## 📚 Test Types

### 1. **Smoke Test**
Quick test with minimal load
```bash
k6 run scripts/smoke-tests.js
```

### 2. **Load Test**
Gradual increase of users
```bash
k6 run scripts/load-tests.js
```

### 3. **Stress Test**
Continuous increase until breaking point
```bash
k6 run scripts/stress-tests.js
```

### 4. **Spike Test**
Sudden traffic spike
```bash
k6 run scripts/spike-tests.js
```

## 💡 Example K6 Script

```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '30s', target: 20 },  // Ramp up
    { duration: '1m', target: 50 },   // Stay at 50 VUs
    { duration: '30s', target: 0 },   // Ramp down
  ],
};

export default function () {
  const res = http.get('https://httpbin.org/get');
  check(res, {
    'status is 200': (r) => r.status === 200,
  });
  sleep(1);
}
```

## 📊 Key Metrics

- **VUs** - Virtual Users (concurrent users)
- **RPS** - Requests Per Second
- **Response Time** - Time to receive response
- **Error Rate** - Percentage of failed requests
- **Throughput** - Total data transferred
